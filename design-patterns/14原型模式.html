# 自我总结

原型模式是一种创建型设计模式，使你能够复制已有对象，而又无需使代码依赖它们所属的类。

# 使用场景

  • 将一个类作为原型，然后复制出来另一个类。

# 优缺点

优点

  • 减少了子类的构造。原型模式是Clone一个原型而不是请求工厂方法创建一个，所以它不需要一个与具体产品类平行的Creator类层次；

缺点

  • 每个类必须配备一个克隆方法，而且这个Clone方法需要对类的功能进行通盘考虑。这对全新的类来说不是很难，但对已有的类进行改造时，不一定是容易的事。

# 变化点&稳定块描述

原型针对的是复杂类的创建，变化或者不变，都是类内部的。

# 数据结构

类继承ICloneable接口并实现Clone方法

# 代码示例

深浅复制：https://www.cnblogs.com/chenwolong/p/MemberwiseClone.html 浅复制：在C#中调用 MemberwiseClone() 方法即为浅复制。如果字段是值类型的，则对字段执行逐位复制，如果字段是引用类型的，则复制对象的引用，而不复制对象，因此：原始对象和其副本引用同一个对象！

深复制：如果字段是值类型的，则对字段执行逐位复制，如果字段是引用类型的，则把引用类型的对象指向一个全新的对象！

代码例子

public class Color : ICloneable
{
    public int Red { get; set; }
    public int Green { get; set; }
    public int Blue { get; set; }

    public Person Person { get; set; }

    public Color(int red, int green, int blue, Person person)
    {
        Red = red;
        Green = green;
        Blue = blue;
        Person = person;
    }

    public object Clone()
    {
        // 浅复制
        return this.MemberwiseClone() as Color;

        // 深复制1
        var color = this.MemberwiseClone() as Color;
        color.Person = new Person("bb");
        return color;

        // 深复制2
        MemoryStream memoryStream = new MemoryStream();
        BinaryFormatter formatter = new BinaryFormatter();
        formatter.Serialize(memoryStream, this);
        memoryStream.Position = 0;
        return (Color) formatter.Deserialize(memoryStream);
    }
}

public class Person
{
    public string Name { get; set; }

    public Person(string name)
    {
        Name = name;
    }
}

/// <summary>
/// 通用的深复制方法,继承该类和ICloneable
/// 类需要定义[Serializable]
/// </summary>
/// <typeparam name="T"></typeparam>
[Serializable]
public class BaseClone<T> : ICloneable
{
    public virtual object Clone()
    {
        MemoryStream memoryStream = new MemoryStream();
        BinaryFormatter formatter = new BinaryFormatter();
        formatter.Serialize(memoryStream, this);
        memoryStream.Position = 0;
        return (T) formatter.Deserialize(memoryStream);
    }
}
